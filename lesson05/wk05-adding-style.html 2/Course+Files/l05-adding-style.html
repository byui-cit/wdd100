Step 01 

First review the siteplan wireframe to help remember what we are trying to do. Notice that there are several elements on the page that are related and grouped. On this wireframe I added red boxes to help to see some of those relationships. We need to add some markup to make 'boxes' around those same elements in our html.

Html has several tags specifically for this purpose. Some have specific semantic meanings and others don't. We should always try and use the ones with meaning whenever we can. Here is a list of some of the more common of these tags: <header>, <footer>, <main>, <section>, <nav>, <article>, <aside>, <div>, and <span>

All of them behave the same in the browser except for <span>, but they are used to show different sections of the page. We are going to use the more specific ones first, <header>, <nav>, <section>, <footer>. 

We always try to use semantic tags first before we resort to using the <div> tag.  The <div> is a genaric statement that should always be used with with an id or class only , and only used if none of the semantic tags seem to apply to the content you are boxing in.

Step 02  

Let's add some boxes. First notice that the links are all related. Let's put them in a box. One of the tags I listed above is <nav> it is specifically for designating navigational elements on a page. Let's wrap the list <ul> </ul>that contains our navigation in a <nav></nav> element.

Step 03

Next,  notice on the wireframe that there are sections consisting of related headline, paragraph, and image. Let's group each of those and designate them as <section>s.

Here is an example of how one of those sections should look:  <section> .  Repeat for each section.  Remember that section 3 does not have an image.

Section Example

Each of your sections will look like this:
<section>
    <h2>We run Rivers to relax.</h2>
    <p>We believe that there is nothing like a river trip for relaxing...  </p>
    <img src="images/picture1.jpg" alt="river-picture">    
</section>

Step 04

That's enough boxing for now. Let's add some styles. In the <head> </head> portion of our page let's add a <style></style> element.

Step 05

The wireframe has a rectangle around the navigation menu. This usually indicates a colored box in that area. Let's add some color to our <nav>. In the <style> block we added let's make a CSS rule modifying our <nav>.

Every CSS rule is made up of 3 elements. First we always need a selector, in this case we want to modify any <nav> elements on our page so our selector will be: nav. Next there are always curly braces { }. Inside those braces we place one or more pairs of 'property to change: value to change it to'

So in this example to change the background color of our <nav> after adding the new rule our style block would look like this: ( you should use one of your selected colors from your color theme ...... probably not salmon ; )


<style>
     nav {
         background-color: salmon;
     }    
</style>

It's worth taking a moment to talk about what valid options you have to designate color. The 3 main ways are hexadecimal, rgb, and predefined color names. Take a moment to visit W3Schools: CSS colors to get a brief outline about the differences. Also take a look at this hat your browser will understand.

  • hexidecimal
  • rgb
  • predefined color names  
Before you make a choice of which system you will use, visit all three to see the benefits of each.

Step 06

The color you changed the background of the <nav> to may have made your text hard to read. In the site plan you should have chosen a color for your links. Add another line to your css rule to change the color for the text in the <nav> as well. Again you should use a color from your chosen palette.

Pick something that looks good with the background color and that is easily readable. 

Note:  If your background is a dark one, a lighter text should be used and likewise, if you have a light background color, use a dark text color.

After you add the text color, your code should look something like this:


<style>
     nav {
         background-color: salmon;
         color: #331100;
     }    
</style>

Step 07

Our next step is to add the background color to the second section on our page.

We have a slight problem here though. If we use the <section> as a selector like we did with <nav> it will change all of our <section>s , which we don't want. From the Khan Academy lesson you should have learned about 2 other selectors: id selectors and class selectors. Id selectors should be used for elements that have unique styling needs, class selectors for those elements that might share styles with other elements.

In this case let's use an ID. Add an id="" to the second section so we can apply some styles just to that one. keep it simple but descriptive, like <section id="sec2">.   Next,  add a css rule to change the background color of that box to one of your colors. (Remember that when you write the rule you need to add a '#' to your id name so the browser knows it's looking for the id and not the tag)  Now, give each of the other sections an id="name" to make it so that later we can add rules to them if we choose to do so.

Step 08 

Next,  let's make our headlines stand out a little more. Lets change the color of all headlines again to one of the colors in your palette. Since we need to make sure everything from <h1> to <h6> gets the same styling we will take advantage of the fact that you can apply the same styles to more than one selector, so we can write one rule instead of six to change them all.

To have a set of rules apply to more than one selector we separate them with a comma and a space after each comma. So the rule to change the color of all headlines would look something like this:

Multiple selector Example


h1, h2, h3, h4, h5, h6 {
     color: #6495ED;
}

Step 09

Changing the color of the second section may have made the text there hard to read. For our final step lets use a class selector to change the color of that text only. Add a class="" to the <p> inside the second section like this: <p class="  ">. Choose a very short descriptive name for the class, like class="pspec" (paragraphspecial) and then create a css rule to change the color. Remember that you need to precede your rule with a '.' so that the browser knows it's looking for the class and not a tag.  If done correctly it should only change the text of the paragraph we applied the class to.
